# Forgebook Contract

Forgebook is notebook-first. Jupyter notebooks are the source of truth; publishing metadata lives outside notebooks in `registry.yaml`.

## Registry rules

Required fields:

- `slug`: lowercase letters, numbers, and hyphens only; pattern `^[a-z0-9]+(?:-[a-z0-9]+)*$`
- `path`: starts with `notebooks/` and ends with `.ipynb`
- `title`: display title
- `authors`: at least one author with `github`

Optional fields:

- `description`: max 500 characters
- `date`: `YYYY-MM-DD`
- `tags`: each tag must exist in `tags.yaml`

Author rules:

- Each `authors[].github` value must exist in `authors.yaml`.
- Each author profile must include `name` and `title`.

Tag rules:

- Tags are curated.
- Unknown tags fail CI.
- Suggest new tags via PR rather than inventing them silently.

## Notebook rules

- Do not put site-specific metadata inside notebooks.
- Do not start the notebook with `# H1`; the page title comes from `registry.yaml`.
- Use Python 3.10+ compatible code.
- Keep notebooks runnable top-to-bottom.
- Keep setup in one early cell when possible: clients, helpers, output dirs. No business logic in setup.

## Media and data rules

Organize assets by recipe slug:

```text
notebooks/media/<slug>/01-screenshot.png
notebooks/data/<slug>/sample.csv
```

Reference from notebooks with relative paths:

```python
df = pd.read_csv("data/<slug>/sample.csv")
```

```markdown
![Project home](media/<slug>/01-project-home.png)
```

Do not hardcode site URLs or `/forgebook/` paths in notebooks.

## Site rules relevant to review

- Internal site links in Astro code use `withBase()` or `import.meta.env.BASE_URL`.
- `site/src/data/popular.json` is generated by CI; never edit it manually.
- The notebook renderer rewrites relative notebook media paths to the correct site base path.

## Validation commands

For registry changes:

```bash
cd scripts && npx tsx validate-registry.ts
```

For site changes:

```bash
cd site && npm run build
```

For notebook execution, use a clean environment when practical:

```bash
jupyter nbconvert --to notebook --execute notebooks/<recipe>.ipynb --output /tmp/<recipe>-executed.ipynb
```

Use these commands as evidence. If you cannot run them, state that the verdict is based on static review only.
